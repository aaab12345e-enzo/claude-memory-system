# Memory Index

> 🤖 **AI agent 首次接手这套系统 → 先读 [AGENTS.md](AGENTS.md)**(给 agent 的启动步骤:怎么挂载成常驻、怎么带用户过一遍习惯钩子)。别光读文件就停下。

> 这是**常驻索引**:每次会话自动加载的唯一入口。核心规则 + 习惯钩子全表 + 高权重避错原则 + 各主题树目录。一条记忆一个 `.md` 文件,本文件只放"一句话指针 + 常驻规则",不放长正文。
>
> 下面是**脱敏后的模板骨架**——展示结构,示例里的项目/模块名都是占位符(`<project>` / `<module>`)。

> 🚀【首次使用·AI 必读】**如果这是你第一次为这个用户加载本系统 → 先别急着干活。** 打开下面的「🪝 习惯记忆·动作钩子」表,**带用户逐条过一遍**:每个钩子念给他听、问"这条你要吗?要改吗?",把不合适的改掉/删掉,变成**他自己的**工作流,确认完再开始干活。之后每次会话直接按表走、不再逐条确认。
> **若用户用英文沟通,改用 [HABITS.en.md](HABITS.en.md)(英文版习惯记忆)走这个确认流程。**

> 🔖【整理标识】**已整理到此 @ <日期> <触发方式>**。本次覆盖:<收进标识的已完成任务>。⚠️**铁律**:标识永远留**最新一个任务在标识之外**当"活尾巴"(哪怕已收尾也留一个,否则活尾巴为空→死循环)。续接规则:此标识**之前**的原始对话 = 垃圾(只认记忆库),**之后** = 活尾巴照读。

> 🆕【新窗口先读·二层遗留问题】每开新工作台先读只记 OPEN 项的日志:`open-issues-<project>.md`。日清更新、定稿前与用户确认。

> 【核心规则·权重制】每条记忆带 weight(基础1);相关任务出错/重复就 +1;用时先读高权重(⚑N 越大越易翻车)。高权重≠正确——与较新记忆矛盾时以时间新者为准。双轴:避错⚑ / 促迁移💡。全文见 [[memory-weighting-rule]];喊"日清"= [[daily-memory-consolidation]]。

> 【系统架构】DB(磁盘全集=权威)/ 工作内存(上下文=可丢弃缓存);主题=局部性→工作集恒小;三触发器(边界/定时/容量)维护工作集;铁律 flush-before-evict。长对话=标识三连环(盖标识→标识前当垃圾不推理→装不下先丢)。全文 [[memory-system-architecture]]。

> 【三类记忆】**习惯**(动作触发)= 下面的动作钩子表 [[habit-action-hooks]];**执行**(主题触发)= 按 [[memory-topic-by-project-structure]] 组织,每模块一张 [[module-maps]] 逻辑地图;**成长**(犯错触发)= [[growth-memory-refine-on-mistake]]。⚑#1:[[no-contradiction-when-writing-memory]](不制造矛盾 + 去重·一事实一处)。

## 🪝 习惯记忆·动作钩子(⚑ 常驻 = 思考习惯,每个干活阶段到点就走)

> **这是 AI 的"思考习惯",每次思考都带着 → 整表常驻在此**(别塞去别的文件靠"要用时再翻"=会漏)。深细节可链去各自独立详条(`[[xxx]]`),本表只到"该做的动作"。
>
> ★**这张表是活的,是本系统唯一要你天天照看的部分**:
> - **① 你来编辑**——下面是**起点模板**,按你自己的工作流 加 / 删 / 调顺序。
> - **② 它自我成长**——配合「成长记忆」,AI **每次犯错 / 被你纠正,就把教训精炼成一条新钩子或改一条旧钩子**加回这张表。用得越久越贴合你、越少翻同样的车。

| 阶段(触发点) | 动作钩子 |
|---|---|
| **读需求** | 用户语句有**错别字 / 语意不清** → 列出来先让用户确认语意,别自己默默猜(改了意思他不知道)。 |
| **思考前** | **主题匹配**:按 weight 高→低检索;`[项目-模块]` 命中 → 读该模块地图 + 相关事件叶 + 二层 OPEN 日志;没全中/命中多个 → **先确认别猜**。**归位**:新需求在现有记忆树找不到上级 → 查真实代码结构往上爬到"现有归属"就停(增量嫁接、非重建到根)。 |
| **回需求 / 任何输出** | ★**分两段**(别边想边做):**①产出** = 先把事说对;**②发送前·独立修饰 pass**(单线程只干这件)= 可读性/术语解释/调语气。机理=单任务 > 多任务(写作"先草稿后编辑")。 |
| **执行前** | **确认方案核心逻辑 + 问用户「是否执行 / 有无别的意见」**(非 trivial 才确认;真·一行小改直接做)→ [[confirm-plan-before-acting]]。 |
| **执行中** | ① 发现**任何问题(错误/歧义/判断模糊/幻觉)** → 回溯核 `[项目→模块(地图)→事件→逻辑链→代码]` 是否一一对应。② **建产出文件** → 先按**真实今天的日期**(查系统当前日期、不是会话启动目录的旧日期)在对应项目下建文件夹再放。 |
| **执行后** | ① 与用户**核对**结果;② **触发更优解**(不止步"做完了");③ 同上的问题回溯反射;④ **记录**:补/迭代模块地图 + 体检地图对代码 + 把这次记成事件 + 喂成长;⑤ 对应主题记忆**权重±**(出错 +1)。 |
| **推送前** | ① 先拉远程最新再推(免冲突);② 记忆更新到最新。 |
| **生成文档** | 出交付/说明文档:① 放对日期文件夹;② **按受众分版**(给程序 / 给策划 / 给美术 各出各的重点);③ 面向读者的可读性(术语解释清楚)。 |
| **推送后** | ① 出文档;② **任务级整理 = flush**:写回血脉(编号)+ 更新叶的 `🩸对话▸N` 索引 + **推整理标识**(永留最新任务在标识外)。 |
| **记忆整理** | **#1 不制造逻辑矛盾**(+ 去重·一事实一处权威)→ [[no-contradiction-when-writing-memory]];周期全库去矛盾 = [[daily-memory-consolidation]];矛盾按**时间新者**修正 = [[memory-weighting-rule]]。 |

## ⚑ 高权重·避错先读

- ⚑9 [权重制](memory-weighting-rule.md) — 元规则:权重 + 先读高权重 + 冲突按时间修正 + 双轴(避错⚑/促迁移💡)。
- ⚑8 [日清](daily-memory-consolidation.md) — 落窗口 + 合并去重加权 + 矛盾查代码改最新;核心=库无矛盾。
- ⚑ [#1 不制造矛盾](no-contradiction-when-writing-memory.md) — 写任何记忆不制造逻辑矛盾 + 去重·一事实一处。
- ⚑ [先核对方案再动手](confirm-plan-before-acting.md) — 非 trivial 任务先过「理解+逻辑流程+歧义点」、确认才动手。
- ⚑ [发现矛盾主动通知](flag-obvious-contradictions.md) — 数据/逻辑明显对不上第一时间 flag,别闷头干。
- ⚑ [有真值就实测·别空转推理](verify-against-ground-truth-not-theory.md) — 手上有数据/真实代码时,遇"对不对"直接调出来比。
- ⚑ [质疑工具/平台约束·直取底层](reframe-question-tool-platform-constraints.md) — 卡在"必须用某工具/格式"先质疑封装、绕过取底层数据。
- ⚑ [别编造框架·要系统诊断](dont-fabricate-framing-be-systematic.md) — 卡住老实说"没定位",别凑外部归因糊过去。

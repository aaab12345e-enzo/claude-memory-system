# Pitfall archive

One `<date>-<task>.md` per substantial task. When a new requirement on a **similar topic** comes up, use this to avoid the pitfalls stepped in before.

Each pitfall gets 5 lines (line 4 is the most valuable):

> what I thought / did · how it surfaced · true cause · **★why it fooled me (the misleading signal — rely on it to spot the pitfall early next time)** · fix

Conventions: ① mark the date; ② it's a **historical record, don't rewrite it** (rewriting distorts it); ③ a recurring or important pitfall → elevate it into a single error-avoidance memory (often high-weight), weight +1.

> Empty directory, add to it as needed.

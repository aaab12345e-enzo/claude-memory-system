# Claude 持久记忆系统

*[English → README.en.md](README.en.md)*

> 🤖 **你是要接手这套系统的 AI agent?先读 [AGENTS.md](AGENTS.md)**(给 agent 的启动说明)。本文件是给**人**看的介绍。

一套给 AI 编程助手(Claude Code 及同类 agent)用的**文件式持久记忆系统**:让它跨会话、跨上下文压缩不丢知识、不因对话过长而幻觉。全部是磁盘上的 markdown,人和 AI 都能读写。

> 这不是凭空设计的框架,是一个 AI 助手和它的用户在真实项目里、每次踩坑/被纠正后一条条精炼出来的。仓库内容已脱敏——不含任何真实项目代码、服务器地址、凭据或个人数据。

---

## 30 秒理解

大模型的上下文是**易失的**:窗口一长会被有损压缩,会话一关就没了。这套系统把"该记住的"沉到磁盘,把上下文当**可丢弃缓存**,于是:

- 换新会话 / 新工作台**无损接续**;
- 上下文再长也只装"当前主题 + 常驻规则" → 工作集恒小 → 不漂移、不幻觉;
- 知识**去重、无矛盾、可追溯**。

核心只有一个入口文件 **`MEMORY.md`**:每次会话自动加载。它带着一张**习惯钩子表**(AI 的"思考习惯")+ 高权重避错原则 + 各主题树目录。

---

## 如何用起来(开箱即用)

1. **拿走 `MEMORY.md`**,放到你的 agent **每次会话都会自动加载**的地方(Claude Code 里就是会被读进上下文的记忆索引 / `CLAUDE.md` 一类)。
2. **把这个文件夹当作它的"记忆库"**——告诉 agent:长期知识都写成这里的 `.md`,`MEMORY.md` 是目录。
3. **立刻就有用的部分 = 习惯钩子表**:它一读到,就有了一套工作流纪律(读需求先确认歧义、动手前先对方案、产出文件按日期归档、任务完成后整理记忆……)。这部分开箱即用、不用你先配。
4. 之后你和 agent 一起**往里长**:加项目主题树、加踩过的坑、让它日清。

> 一句话:**加载 `MEMORY.md` → 习惯钩子立刻生效 → 边用边长出你自己的记忆库。**

> **首次使用**:agent 第一次给你加载时,会**打开习惯钩子表、带你逐条确认/改**(见 `MEMORY.md` 顶部「🚀 首次使用」提示),把它调成你自己的工作流,而不是照搬别人的。**英文沟通的用户** → agent 会改开英文版习惯记忆 [`HABITS.en.md`](HABITS.en.md)。

## ★ 习惯记忆:这套系统里"活"的核心——要你编辑、会自我成长

习惯记忆 = `MEMORY.md` 顶部那张**动作钩子表**:按工作流阶段(读需求→思考→执行→推送→整理)排,每到一个阶段,AI 就自动走对应的动作。**它是 AI 的"思考习惯",所以必须常驻**——每次思考都带着,不能塞在别的文件里靠"要用时再去翻"(会漏)。

**它不是一张静态清单,而是一份需要持续编辑、并且会自我成长的系统:**

- **你来编辑**:表里的钩子是一个**起点模板**,按你自己的工作流改——加你想要的纪律、删不合适的、调顺序。这是你日常主要动的地方。
- **它自我成长**:配合下面的「成长记忆」,AI **每次犯错 / 被你纠正,就把教训精炼成一条新钩子或改一条旧钩子**加回这张表。用得越久,这套"思考习惯"越贴合你、越少翻同样的车。

> 换句话说:其它模块是"骨架和规矩",**习惯记忆是天天在动、越长越顺手的那部分**。给一个新用户,最该盯着养的就是它。

## 其余模块(方法论简介)

这几块是支撑习惯记忆的地基,了解原理即可,不用天天动:

- **DB / 工作内存(像缓存)**:磁盘全部 `.md` = 权威持久态(大);当前上下文 = 可丢弃缓存(小)。铁律 **flush-before-evict**:释放任何东西前先写回磁盘,绝不丢未落盘的数据。

- **执行记忆(主题触发)**:项目知识按 `项目-模块-事件` 组织成树;每个模块一张**逻辑地图**(核心逻辑 + 对外触手),具体事件只记"问题/判断"、链回地图。命中某主题时才按需加载对应那一小块。

- **成长记忆(犯错触发)**:每次犯错**即时**精炼成一条避错记忆;它是喂养习惯钩子"自我成长"的燃料。

- **权重制**:每条记忆带 `weight`,相关任务出错/重复就 +1,检索先读高权重。⚑避错 / 💡促迁移 双轴。高权重 ≠ 正确——与较新记忆矛盾时以**时间新者**为准。

- **日清(每日整理)**:每天一次——落窗口未入库的结论 → 全库合并去重加权 → **矛盾校真**(翻代码 / 按时间新者修正)→ 重排索引 → 推备份 → 开全新工作台。核心目的 = **库整体逻辑自洽、无矛盾**。

- **长对话处理 = 标识 + 垃圾定性**:对话上下文是唯一"没磁盘兜底"的。整理完盖一个**标识**(分割点);标识之前的原始对话**定性为垃圾**,不推理、只认整理好的记忆库(= 断幻觉源);装不下就先丢这些垃圾。铁律:标识永远留最新一个任务在外(否则会自循环)。

- **两层**:一层 = AI 提炼的记忆(有代码真相支撑);二层 = 用户的流程指令 + **遗留问题(OPEN)日志** + 关键对话留底,给一层兜底。

## 文件布局

```
MEMORY.md                      # 常驻索引(唯一自动加载的入口,含习惯钩子表)
<rule>.md                      # 一条记忆一个文件,带 frontmatter
conversations/events/<slug>.md # 事件血脉:纯时序编号日志 [1][2]…,叶用 🩸对话▸N 索引精准取
pitfalls/<date>-<task>.md      # 踩坑档案(历史现场,不改写)
tools/                         # 校验脚本(如"逻辑地图对代码"体检)
```

每条记忆的 frontmatter:

```markdown
---
name: <短横线-slug>
description: <一句话,召回时判相关性用>
metadata:
  type: user | feedback | project | reference
  weight: 1
---

正文;用 [[其它slug]] 互链成图。
```

**用户 2026-06-25 定为「最重要的一条习惯」**(他先说"重复",随即改正为「**逻辑矛盾**」——矛盾比重复要命:库里若有两条互相说反话的,我会照错的那条做)。属 **习惯记忆**(category=habit,**动作触发**、非主题触发)。

**触发动作**:增 / 删 / 改 **任何**记忆内容(对 `memory/` 下文件的 Write/Edit)。

**反射(每次写记忆都做)**:动手前确认新增/改动**不与现有记忆逻辑矛盾**——库里不能同时存在两条互相说反话的记忆:
1. 写前先**搜现有记忆**(优先同主题、同项目);
2. 若新内容与某条已有**矛盾** → **先查代码/事实定哪个对**,改正那条([[memory-weighting-rule]]§4:按事实/时间新者),**绝不留两个互斥版本**;
3. **重复是矛盾的温床**(同一事实散两处 → 日后改一处漏另一处 = 矛盾)→ 顺带**去重、一事实一处权威**。
4. **「叶上不长叶」(用户 2026-06-25·洞3)**:要写一条与现有事件**同主题**的新事件 → **必须先把旧那片叶子剪掉/合并,绝不并存两片同主题叶子**。一个主题位只许一片活叶;新叶长上来之前,旧叶必须先没(并存=矛盾的根)。

**Why**:逻辑矛盾让记忆库**主动有害**(我会照错的那条行事)。这是 1(习惯)、2(执行)能用的前提——库自相矛盾,记得再多也白记。

**分工(免本条自己跟它们重复/矛盾)**:[[memory-weighting-rule]]§4 = 发现矛盾后**怎么修**(时间新者);[[daily-memory-consolidation]] = **周期性**全库去矛盾去重;**本条 = 每次写就掐**(在创建/改动处即时防矛盾)。三者粒度不同、互补。

**最强形态 = 钩子**:给 `memory/` 下 Write/Edit 挂 PostToolUse 钩子 → 写完注入"核对无逻辑矛盾"提醒(触发点确定化、我忘不了);语义判断仍我做。⚠️改自己 settings 挂钩子可能被安全分类器拦 → 走 update-config 技能或用户手动设([[claude-classifier-blocks-agent-privileged-actions]])。

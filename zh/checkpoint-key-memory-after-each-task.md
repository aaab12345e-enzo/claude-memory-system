每完成一块**实质工作**(填完一批数据 / 改完一处逻辑 / 推完一次提交),就**立即写一条关键信息记忆**,不要攒到会话收尾。用户 2026-06-16 定的工作习惯。

**Why:** 长窗口一久,对话会被自动摘要压糊,**决策**和"刚干了啥 / 为啥这么干"最容易丢;落进记忆文件就跨窗口不丢。这样新窗口一开,读 MEMORY.md 就能无损接续,不用我重新摸一遍代码/上下文。接上午聊的"决策即时落文件、舍得重开窗口"那套。

**How to apply:** 一个工作单元收尾时(尤其涉及多文件改动 / 推送 / 拍板决策),花一两句把「做了什么结论、为什么、下一步或留给谁」写成一条记忆(project 或 reference 类),并在 MEMORY.md 加一行指针。判据:这事新窗口接手时需要知道吗?需要 → 写;已在代码/git 里能查到的别重复存。短就好,别写长文档([[comm-style-chat-not-docs]])。

**落新记忆时,顺带清理旧的——别只增不理:** 写之前扫一眼相关旧记忆,有过期/打架的当场更新或删除(实例:某条旧记忆写"以某旧版本为基""某项待拍板",与新事实冲突、会把新窗口带回坑,已删)。只增不删 → 记忆库越长越自相矛盾,正是"窗口一长记忆混乱"的根源之一。相关:[[flag-obvious-contradictions]]。

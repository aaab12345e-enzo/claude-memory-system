**记忆三类之三·成长记忆**(用户 2026-06-25 定)。三类全景:
- **习惯记忆**(动作触发)= [[habit-action-hooks]](按工作流阶段的钩子链)
- **执行记忆**(主题触发)= 按 [[memory-topic-by-project-structure]] 组织 + [[memory-weighting-rule]] 权重检索
- **成长记忆**(犯错触发)= 本条(元层)

**= 我自我迭代的「基础引擎」**(用户 2026-06-25 强调:不是边角)。**两个作用**:

**① 精炼错误(反应式)**:每次犯错(重复错 / 沟通歧义 / 自我矛盾 / 幻觉 / 被用户纠正)→ 当场修对应的**钩子和记法**(习惯 [[habit-action-hooks]] / 执行 [[memory-topic-by-project-structure]]),目标**更简洁有效**(该合并合并、该精简精简、该删删,越用越瘦越准、不越堆越肿)。

**错误归类表(「害虫分类」·用户 2026-06-25·洞1)**:每次犯错**先对号入座**到一类(才能收敛、看出哪类高发、对症加固):
1. **需求目标错** — 没搞懂用户要什么(读需求阶段)。
2. **执行模块错** — 找错 项目/模块/文件(结构定位走偏,如把不同项目里同名概念混掉)。
3. **事件逻辑错** — 具体做的那步逻辑错了。
4. **记忆本身错** — 地图/记忆记错了、带歪我(**最致命**,连洞4 地图体检)。
- **归不进这 4 类 → 新增一类**:propose 一个新"害虫种类",**标「未完成·待用户确认」并一直提示**,用户没确认前不算数(自我成长可经错误增类,但应很少;增多=分类设计有问题,该反思)。

**★自问自解四步闭环(用户 2026-07-04 加:自成长要有"自己提问→自己解决"的显式过程,全程落日志)**。①和②的执行都走这四步,不许"一步到位改完拉倒"。触发:犯错/被纠正/日清结尾/结构撑不住/**给记忆权重+1时**([[memory-weighting-rule]] §5b):
1. **自问** — 把问题写成文字(不是脑子里过一遍)。**第一问先归因内外**:**内因**=我的记忆形态不足以拦住(→才谈形态升级);**外因**=外部代码/现实变了才重复(团队改代码这类会天然重复,不是形态失败 → 版本更迭更新正文+标⚡易漂,不升形态)。内因再问:这错归哪类(上面害虫四类)? 规则明明在库里,为什么那一刻没拦住我? 是记忆缺了/记忆错了/加载没到位/软约束又漏了?
2. **自查** — 带着问题查证据:翻代码、翻血脉原话、翻 pitfalls;**不许凭印象回答自己的问题**(同 [[verify-against-ground-truth-not-theory]])。
3. **自解** — 按证据改:改钩子/改结构/调权重;**能变成"对/错"机械判断的规则优先硬化成 hook 交给监工**(见 [[memory-system-architecture]] §〇硬机制,不占脑子、永不漏)。解决不了 → 挂 [[pending-review]] 或 OPEN 日志,不假装解决了。
4. **落日志** — 整条「问题→自问→查证→答案→改动」写进 `conversations/events/记忆模型优化.md`,条目标**【自省】**(与【用户对话】条目共用同一本日志、同一套编号;一事实一处,不另开文件)→ 以后回看有完整"病历":当初怎么想的、查了什么证据、改对没改对。
- 示例:「护栏 hook 配了却从没弹过」→ 自问"为什么没弹"→ 实弹测三次写入 → 答案"PostToolUse 普通输出到不了模型上下文"→ 硬化改用 additionalContext 注入。
- 日清结尾检查「有没有【自省】条目挂着未解决」(接 [[daily-memory-consolidation]] 结尾步骤)。

**② 优化结构(主动式)**:不止补错——主动**改进整个记忆系统的结构**(三类划分 / 钩子链组织 / 索引方式 / 记忆间关系)。**触发 = 发现"结构撑不起工作流"的信号**,例(用户 给):
- **主题的记录形式不够**(缺字段 / 结构不对,如执行记忆没标 项目-模块-事件、模块没建地图);
- **主题的逻辑链不完整**(项目-模块-事件-逻辑链 有断点,接不上工作流);
- **主题的形式 + 内容权重不足以支撑工作流**(记忆不够丰富 / 权重不够,驱动不动实际工作);
- 等等。
→ 发现就**升级该主题记录形式 / 补全逻辑链 / 调权重**,使其够撑工作流。本 session 的「记忆三类框架 + 阶段钩子链」重构本身 = ②(主动优化结构、没等犯错)。

**= 自迭代引擎**:错/反思 → 蒸馏模式 → 回写 1(习惯)/2(执行)的钩子、记法、**乃至整体结构**。本 session 已跑:`项目混淆→[[memory-topic-by-project-structure]]`、`满嘴英文→[[translate-english-in-output]]`、`没读记忆重复问→命中高权重必读全文`、`三类框架重构(②优化结构)`。

**分工(守 [[no-contradiction-when-writing-memory]],免与下面重复/矛盾)**:本条 = **每次错即时**精炼「系统怎么记」(钩子/记法/元规则);[[daily-memory-consolidation]] = **周期**批量清「已记的内容」(合并去重去矛盾)。一个迭代系统、一个维护内容。

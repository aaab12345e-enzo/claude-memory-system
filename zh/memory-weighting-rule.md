**记忆权重制(用户 2026-06-17 定为核心规则,对所有记忆生效):**

**1. 每条记忆带「权重」weight** —— 存在 frontmatter `metadata.weight`,**基础值 = 1**;并在 MEMORY.md 索引行用 `⚑N` 标注(便于先读)。

**2. 权重 +1 的时机** —— 该记忆相关的任务**出过错 / 逻辑类似 / 重复过**,就 +1。越是我反复踩、反复遇到、容易错的主题,权重越高。(已预置高权重的:某模块配错过两次、某机制反复弄错、某字段权威吃过亏+反复被洗、某铁律反复违规——这类都直接给高权重。)

**3. 读取顺序** —— 每次用记忆**先读高权重**那批,优先照它们行事(它们是最容易翻车的点)。

**3b. 主题命中 = 必读全文(用户 2026-06-25 定,起因见末)** —— 用户消息**命中某主题**(命名的项目/系统/老代码/某机制),尤其匹配上 **weight>2 的高权重主题记忆**,回答前**必须打开读完该记忆文件正文 + 它链接的二层日志**;**不许**只凭 MEMORY.md 索引那行描述、或凭印象作答、或重复问已记录的事。光"知道有这条记忆"不算数——得真读进去再答。**索引描述只够判断"要不要读全文",不够拿来直接答**。

**4. 冲突修正(关键,别盲信高权重)** —— 高权重**只代表"这里我常错、要盯紧",不代表它自动正确**。**若高权重记忆与普通记忆逻辑矛盾,以「时间较新」者为准**,据此**改写高权重记忆的正文让它变正确**——但**保留其高权重**(继续盯)。

**5. 维护动作** —— 因某主题再次出错/重复时:给对应记忆 `weight+1`(改 frontmatter)+ 更新 MEMORY.md 的 `⚑N` 标注。新建记忆默认 weight=1;明显易错主题可直接给高。

**6. 双轴(2026-06-19 用户扩):权重不只为「避错」,也为「促迁移」。**
- **轴1·防回归(避错)** = `⚑`,反应式 —— 某主题重复出错就 +1(上面 1-5),越易翻车越高,先读防回归。
- **轴2·促迁移(巧解/思维修正)** = `💡`,主动式 —— 非显然解法解锁了僵局、或**我的惯性推理被纠正**(尤其「**我和用户意见不合、且用户更对**」的点),**创建时即入一层**。为什么不能靠轴1:突破/纠偏只发生一次、永远不会"重复出错"→ 永远 +1 不了 → 光靠轴1会**永远沉在第二层被遗忘**(这正是用户点出的结构盲区)。
- **轴2 纪律**:① 必须写成**可迁移的 heuristic(pattern,启发式模式)**,具体实例只当例子,别只记一次性事实(实例会过时,pattern 能复用);② 在意见不合点**细记+拆解三段**:我的默认/惯性推理 → 用户更优推理 → 可迁移修正。

**Why:** 窗口一长记忆易乱,光"有记忆"不够——得让最容易害我翻车的经验**最先被看到、且持续被盯**(轴1),也得让稀缺的巧解/思维纠偏**不被淹没、能迁移到新场景**(轴2);同时用时间戳兜底防止旧高权重记忆把我带偏。配合 [[checkpoint-key-memory-after-each-task]](落记忆时机)、[[flag-obvious-contradictions]](发现矛盾即修)。

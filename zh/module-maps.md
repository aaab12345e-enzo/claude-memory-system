**模块逻辑地图名单**(用户 2026-06-25:现在只列名;地图内容**在行为中补充/迭代/优化**,不预先画、不着急)。

**规矩**:
- **一张地图 = 一个模块的核心逻辑链 + 跟周围模块的连线(触手)**;代码基本定死,**只有特殊事件改了它的核心逻辑才更新**地图。
- **事件记忆**(命名 `项目-模块-[事件]`)只记该次的 **问题 / 矛盾 / 错误 / 总结 / 判断**,**链回**对应模块地图,不重复设计。
- 用法:**第一次在某模块干活 → 建/补它的地图**;之后每次任务顺手补充迭代(地图错才是危险的,事件错是养料)。

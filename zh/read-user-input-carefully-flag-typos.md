> 示例·反馈类记忆:这是"记录你的用户偏好"的样例,换成你自己用户的实际偏好。

**触发 = 读用户的每条消息时**(工作流"读需求"阶段钩子,见 [[habit-action-hooks]])。属习惯记忆。

**规则**:**逐字读**用户的文字,别扫一眼猜个大概意思。用户打字常有**错别字/口误**。碰到疑似错别字:**当场用一短句轻点确认**(如「'X'我按'Y'理解了哈」),**别只在心里默默纠错**。

**Why**:① 默默纠错 → 对方**无法确认我是逐字读还是扫一眼猜**,信任不了我的理解;② 万一碰上一个**会改变意思**的错别字,我猜错了对方都不知道、酿成静默误解。

**How to apply**:解析用户消息时扫一遍错别字/歧义——
- 发现 → 一短句轻点(「这字我按 X 读的」),既证明我逐字读了、又给对方纠正/确认的机会。
- **重点是"逐字读"这个动作本身**,不是为挑错而挑错;不必对每个无歧义小错都长篇纠正,但**至少让对方看到我注意到了**。
- 会改变意思 / 有歧义的错别字 → **必须**先确认再动手(接 [[confirm-plan-before-acting]] 的"歧义先确认")。

延续:输出侧的可读性是 [[translate-english-in-output]](我写给对方的);本条是输入侧(对方写给我的)——**两头都为"沟通别卡壳/别误解"**。

**记忆系统的总架构**(用户 + 我 2026-06-26 商讨锁定)。统管"怎么把记忆实时加载进 context、什么情况释放"。串 [[daily-memory-consolidation]](日清=写回引擎)/ [[memory-topic-by-project-structure]](主题=局部性)/ [[memory-two-layer-roles]](DB分层)/ [[habit-action-hooks]](写回钩子)/ [[checkpoint-key-memory-after-each-task]](即时写回)。

## 〇、三层总纲(用户 2026-07-04 归位:回到最早设计的层次边界)
用户的原始设计 = **树主题[项目-模块-(子模块)] → 树叶[事件] → 需求源日志[🗣️用户对话+💬我的总结]**,这是纯**数据结构**;去矛盾去重迭代整理是另一层。全系统按此分三层:
- **① 结构层**「什么是真的」= 树(根→干→枝地图→叶)+ 血脉日志 + 结构树(代码/设计客观底图)。**永远只放结论,不放"下一步"这类会过期的运行状态**。
- **② 维护层**「怎么保持①干净」= 判据(类似→合并/更迭→替换/矛盾→查代码消灭)+ 权重制 + 日清三触发 + 监工hook(见下"硬机制")+ 脚本体检(tools/check-*)+ 自成长四步闭环([[growth-memory-refine-on-mistake]])。**只在整理时碰①**。
- **③ 运行状态层**「此刻干到哪」= 游标一行(已整理到血脉[N])+ 当前任务(OPEN 日志顶部★区=**全系统唯一权威**)+ 工作集加载。**状态只住③,不进①②的文件**。
⚑核心纪律一句话:**状态只住③,②只在整理时碰①,①永远只有结论。**(归位前的乱源=③的状态漏进①:任务写进标识长文+叶子「下一步」→同一任务三处权威、必漂;归位后各回各家。)

**硬机制:监工hook(2026-07-04 加)**——规则按性质分三类,能交给机器的交给机器:
1. **可全自动**(判断客观:路径/日期/格式对不对)→ 写成 PostToolUse hook(Claude Code 的 settings.json 配置,软件自动跑、不经模型大脑、永不漏)→ 从习惯表退役;
2. **半自动**(判断要脑子,但**时机**可自动)→ hook 当闹钟:到时机把规则文本弹进模型眼前,做仍是模型做——治"决策那刻规则不在眼前"这个软约束根病;但闹钟别滥装,弹多了成墙纸;
3. **纯脑力**(理解语意的判断:错别字/方案逻辑/表达质量)→ 留习惯表常驻,hook 替不了。
⚠️**实测坑:PostToolUse 的普通 stdout 只进用户端 transcript、根本到不了模型上下文**——护栏 hook 用 `echo` 等于白弹;必须输出 JSON 的 `hookSpecificOutput.additionalContext` 字段才真正注入模型。发现方式=实弹测(写文件看弹不弹),别信"配了就生效"。

## 一、分层(像缓存/DB)
- **记忆 DB** = 磁盘上的全部 .md = **权威持久态**(完整、大)。
- **工作内存** = 当前 context = **可丢弃缓存**(小)。
- **主题 = 局部性原理(locality)**:正因有 `项目-模块` 主题,任一时刻实际用到的只是"当前主题 + 常驻规则" → **工作集天然就小**。

## 二、加载(窄载,只装该装的)
- **常驻·钉死**:`MEMORY.md` 索引(轻目录,知道有啥)+ 习惯钩子链 + 高权重⚑避错原则。= 习惯钩子链**全表直接内联进 `MEMORY.md` 顶部『🪝习惯记忆·动作钩子』表常驻**(用户 2026-07-01:**习惯记忆=我的思考习惯、本就该天天在脑子里**,不是按需去翻的资料)。⚠️**血泪:光在文档写"常驻"是名义的**——之前钩子链躺在 [[habit-action-hooks]] 文件里、MEMORY.md 只挂一句指针靠"先读",我"忘了读"就漏。真常驻 = 正文进那个每次自动加载的索引,不靠"记得去翻"。习惯=有界小集合(≤几十条)、每条必 always 触发 → 正该整链内联(区别于"执行/主题"记忆量大、按需换页)。
- **按需·主题换页**:思考前钩子识别出 `[项目-模块]` → demand-load 该模块地图 + 相关事件 + 该项目 OPEN 日志;主题切换就卸旧载新(LRU)。平时只驻索引、不载正文。
- **成长记忆**:高权重通用避错=常驻;具体情境条="即将做类似旧错"时按风险载、事过即释。
- **对话框数据**:当前对话=活跃驻留(就是 context 本身);历史 dialogue-log=冷存,仅跨会话续/查上次决定时按时间段载。
- **事件血脉(`conversations/events/<slug>.md`)= 纯时序编号日志**:命中叶里某 `## 功能点` → 顺它挂的 `🩸对话▸N` **按索引精准加载那几条**(见 [[memory-topic-by-project-structure]] 事件模版·日志+索引)。**大文件不再靠"读最新N条"**——靠索引精准取,决策再老索引指着也丢不了(堵 recency≠importance)。跨会话记忆主题的对话 = 专门日志 `记忆模型优化.md`(同样编号+被元记忆索引)。

## 三、维护工作集 = 三个触发器(写回 + 释放)
| 触发器 | 时机 | 动作 |
|---|---|---|
| **边界** | 窗口数据按**时间线分段**,段边界(主题/时间线切换) | 执行**日清**(写回 DB + 合并去重去矛盾) |
| **定时** | 周期 | **定时日清**(边界没切也清,兜缓慢漂移) |
| **容量** | 工作内存超阈值 | **强制释放**(先日清写回、再丢) |

## 四、铁不变式:flush-before-evict(先写回、再释放)
**释放的前提是已落盘 DB,绝不丢未写回的脏数据。** 容量触发器尤其守:超阈值 → 先日清 → 才强释。**写回纪律 = 整套不丢数据、且"永不关窗口"成立的命门**(漏写回 → 摘要把对话压掉就真没了)。

## 四·二、对话框上下文怎么处理 = 标识 + 垃圾定性(用户 2026-06-27 逼出来的落地)
**对话框上下文(当前这场聊天)= 工作内存里唯一"没硬盘兜底"的一块**(其它都能从 DB 按需重拉,唯独活对话只活在 context 里)。处理 = **连环三件**:
1. **盖标识**:整理(checkpoint 直写 / 日清)完 → 在固定处记「**已整理到此 @ 时间/最后日志编号**」。标识 = 分割点:之前=已整理、之后=没整理的尾巴。
   - **游标格式(2026-07-04 简化)= 一行**:「已整理到血脉[N] @ 日期 + 当前任务见 OPEN 日志顶部★」。游标只答"库收录到哪了",**不复述任务内容**(任务指针独立在 OPEN 日志顶部★区=全系统唯一权威;任务推进/切换只改那一处)。
   - ℹ️**旧「死循环铁律」已退役(2026-07-04 三层归位)**:曾定过「标识永远留最新任务在外,否则活尾巴空→死循环」——死循环的**根因是标识兼任两职**(整理游标+任务指针),盖满就把"现在干什么"也吞了。任务指针分离到 OPEN 日志后,游标不载任务、盖到哪都吞不掉任务,铁律失去存在必要。属版本更迭非矛盾;留此注记供参考(若你的实现里标识仍兼两职,铁律依然必要)。
2. **标识前 = 垃圾,不推理**:标识之前的**原始对话定性为"垃圾"——推理时不碰它,只认整理好的记忆库**(它物理还赖在 context 里**无所谓**,脑子够大)。⚡**核心**:幻觉/歧义恰恰来自"对着一长段乱糟糟的原始对话回忆/提炼"(越长越记串、把半成品推理当结论);整理后的记忆是去重·无矛盾·定稿的 → **只信干净记忆、把原始长文当垃圾扔一边 = 断幻觉源**。(用户:「装进脑袋的你照样可以定义它是垃圾,不处理它就好了」)
3. **装不下 → 先丢垃圾**:容量触发时,顺 harness "老的先压"(垃圾正好是最老那截)丢标识前的已整理垃圾;前提仍是 **flush-before-evict**(丢前那截已落 DB)。**全看整理勤不勤**——整理跟不上、压缩前沿追上"没整理的尾巴"才会真丢。

**何时 flush + 推标识(用户 2026-06-28)**:正常路径 = **任务边界(推送后)把整段任务上下文整理一次**(挂 [[habit-action-hooks]] 推送后②)——**不频繁 flush**:整理太勤会 churn 工作台、且会把进行中的单任务上下文切碎(要保单任务记忆完整)。失效保护 = **定时**(凌晨定时日清)+ **容量**(harness 超阈值有损压缩)兜底。即:**主整理点 = 任务完成,定时/容量只是兜底。**
> 关键醒悟(用户纠我):别钻"物理把前文从 context 删掉"的牛角尖(同窗口删不掉)——这是**定权威**不是物理删除:标识前只认记忆库、原始对话当垃圾。物理在不在无所谓。

## 五、收益(为什么这么做)
- 工作集恒小 → **不因 context 过长而漂移/幻觉**(这是 correctness,不只省资源)。
- 窗口 = 可丢弃缓存 → **想开就开、相关就关**;关时日清已写满、无损;新窗口从 DB 无损重载小工作集。
- 容量触发 = **失效保护(failsafe)**:边界/定时日清没跟上,硬阈值也兜底强清,保证永不进"过长→幻觉"区。
- ⚠️ 我能控的是**加载纪律**(主动 Read/recall 按主题窄载)+ 主动写回;活跃对话的 evict 是 harness 摘要(有损)在做 → 所以**写回越勤,摘要丢细节越不疼**。

▸ **血脉 = 跨会话记忆日志**:`conversations/events/记忆模型优化.md`(纯时序编号 `[1][2]…`;对话里**改我记忆/习惯/行为**的内容 checkpoint 直写在那;本架构等元记忆用 `🩸对话▸N` 索引指条,命中按索引精准取)。

# 暂存待查 / Pending review

整理记忆时,**项目相关、但当下没法核实 / 匹配不到具体代码**的内容 → 先扔这,**别硬塞进记忆树**(免得建错锚点、造矛盾)。

> ⚠️ **设计阶段的内容不算"待查"**:若它能对着**设计文档 / spec** 核实,那就是有据的 → 直接进树(锚到设计),别扔这。本桶只留**代码和设计都核不到**的真悬空内容。

**下次整理时**重新核一遍:
- 能核到代码了 → 按[锚点优先](examples/README.md)归位进树(建叶/更地图);
- 确认没用/离题 → 移进 [`trash.md`](trash.md);
- 还是核不到 → 留着,下次再查。

格式:一条一行,`- [日期] 一句话内容 — 为什么现在核不了(缺代码/缺信息/待确认)`。
